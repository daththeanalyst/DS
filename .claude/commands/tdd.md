Use the `test-driven-development` skill to implement features using the RED-GREEN-REFACTOR cycle.

1. Write a failing test first (RED)
2. Watch it fail for the right reason
3. Write minimal code to make it pass (GREEN)
4. Refactor while keeping tests green (REFACTOR)
5. Repeat for each feature/behavior

Target: 80%+ code coverage. Never skip the failing test step.
